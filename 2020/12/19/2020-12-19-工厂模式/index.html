工厂模式

遇见狂神说

◆核心本质:
◆实例化对象不使用new，用工厂方法代替
◆将选择实现类，创建对象统一管理和控制从而将调用者跟我们的实现类解耦。

◆简单工ㄏ模式
◆用来生产同一等级结构中的任意产品（对于增加新的产品，需要扩展已有代码）

◆工厂方法模式
◆用来生产同一等级结构中的固定产品（支持增加任意产品）

简单工厂模式:

1.定义一个接口

package factory.simple;
public interface Car {
    void name();
}

2.定义一个车工厂类

package factory.simple;
//静态工厂模式
//增加一个新的产品，如果你不修改代码，做不到！
//不是开闭原则
public class CarFactory {
    public static Car getCar(String car){
        if(car.equals("五菱")){
            return new WuLing();
        }else if (car.equals("特斯拉")){
            return new Tesla();
        }else {
            return null;
        }
    }
}

3.特斯拉车

package factory.simple;
public class Tesla implements Car{
    @Override
    public void name() {
        System.out.println("特斯拉");
    }
}

4.五菱宏光车

package factory.simple;
public class WuLing implements Car{
    @Override
    public void name() {
        System.out.println("五菱宏光");
    }
}

5.实现类

package factory.simple;
public class Consumer {
    public static void main(String[] args) {
        //接口，所有的实现类, 工厂
       // Car car = new WuLing();
        //Car car2 = new Tesla();
       //2.使用工厂创建
       Car car = CarFactory.getCar("五菱");
       Car car2 = CarFactory.getCar("特斯拉");
        car.name();
        car2.name();
    }
}

工厂方法模式:

1.接口定义

package factory.method;
public interface Car {
    void name();
}

2.

package factory.method;
//工厂方法模式
public interface CarFactory {
    Car getCar();
}

3.摩拜单车

package factory.method;
public class MoBai implements Car{
    @Override
    public void name() {
        System.out.println("摩拜单车");
    }
}

4.摩拜单车工厂

package factory.method;
public class MoBaiFactory implements CarFactory{
    @Override
    public Car getCar() {
        return new MoBai();
    }
}

5.特斯拉

package factory.method;
public class Tesla implements Car {
    @Override
    public void name() {
        System.out.println("特斯拉");
    }
}

6.特斯拉工厂

package factory.method;
public class TeslaFactory implements CarFactory{
    @Override
    public Car getCar() {
        return new Tesla();
    }
}

7.五菱宏光

package factory.method;
public class WuLing implements Car {
    @Override
    public void name() {
        System.out.println("五菱宏光");
    }
}

8.五菱宏光工厂

package factory.method;
public class WuLingFactory implements CarFactory{
    @Override
    public Car getCar() {
        return new WuLing();
    }
}

9.实现类

package factory.method;
import factory.simple.CarFactory;
public class Consumer {
    public static void main(String[] args) {
        Car car = new WuLingFactory().getCar();
        Car car2 = new TeslaFactory().getCar();
        car.name();
        car2.name();
        Car car3 = new MoBaiFactory().getCar();
        car3.name();
        //总结：
        //结构复杂度：   使用simple简单工厂
        //代码复杂度：   使用simple...
        //编程复杂度：   使用simple...
        //管理上的复杂度: 使用simple...
        //根据设计原则：工厂方法模式
        //根据实际业务：简单工厂模式
    }
}

小结:
◆简单工厂模式（静态工厂模式）
虽然某种程度上不符合设计原则，但实际使用最多！
◆工厂方法模式
不修改已有类的前提下，通过增加新的工厂类实现扩展。
◆抽象工厂模式
不可以增加产品，可以增加产品族！
◆应用场景:
JDK CalendargetInstance中的方法
JDBC中 Connection的对象的获取◆ Spring中lo容器创建管理bean对象◆反射中 Classnewlnstance对象的方法
